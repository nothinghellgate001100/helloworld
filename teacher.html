Project Requirements for Backend Development

1. User Registration and Login System

Registration Requirements:

  • Customers must provide their Gmail address and phone number.
  • Implement a CAPTCHA system for security.
  • Send a verification code to the customer's Gmail to complete registration.
  • Customers must also enter an invitation code to register, which helps track who invited them.
  • Each invitation code should have a limit and be linked to a supervisor's account.
  • I need a total of 10 supervisor accounts. Supervisors should be able to view and manage data of users they invited.

2. Supervisor Account Capabilities

  • Supervisors should have the ability to:
  • View the data of the users they invited.
  • Edit certain aspects of user data such as account balance and deposit details.
  • Display specific orders.
  • Change user information, including Telegram handles.

3. Deposit System

  • When a customer wants to make a deposit, a button should generate a wallet address from the backend.
  • Only the admin should have the capability to change this wallet address.
  • The admin should also be able to create new supervisor accounts.

4. Withdrawal System

  • Withdrawal Requirements:
  • Users must enter the amount and withdrawal password.
  • A notification should be sent to the user's supervisor for approval.
  • Supervisors should verify the user's wallet address and send the withdrawal amount.
  • Wallet Management:
  • Users must submit a front and back photo of their ID, and a face recognition video clip.
  • This information should be visible to the supervisor for verification.
  • Users should select the type of wallet address and paste it.
  • Supervisors must verify the wallet address. Once verified, a "Verified" box should be displayed.
  • The same withdrawal password set by the user will be used for withdrawals.
  • Only supervisors and admins have the authority to verify wallet addresses.
  • Admins should have access to oversee all supervisors' activities and manage their users.

5. Orders System

  • Orders should be displayed as packages that can be allocated to users based on their deposit amount.
  • Users should be shown their total commission and account balance according to their deposit.
  • If a user's order exceeds their deposit amount, they should receive a message to make a deposit before submitting the order.
  • Specific details and a sample link for how the orders should be displayed will be provided later.

6. Default Accounts

  • Create some default accounts to demonstrate how to submit withdrawals, deposits, and orders.
  • Provide examples of these default accounts for reference.

Additional Requirements

  • Ensure robust security measures throughout the system.
  • Detailed documentation of the backend code and functionalities.
